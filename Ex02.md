# Bài 2: Tối Ưu Prompt Sinh Docstring & Giải Thích Logic

## 1. Phân tích lỗ hổng của prompt cũ

Prompt cũ:

```text
Giải thích hàm này làm gì và thêm comment vào code giúp tôi: [Đoạn code]
```

Prompt này chưa hiệu quả vì:

- **Thiếu vai trò**: Không yêu cầu AI đóng vai lập trình viên có kinh nghiệm hay chuyên gia bảo trì mã nguồn.
- **Thiếu mục tiêu cụ thể**: Chỉ yêu cầu “giải thích” và “thêm comment”, chưa nêu rõ cần giải thích theo nghiệp vụ.
- **Thiếu ngữ cảnh**: Không nói đây là hàm tính chiết khấu trong hệ thống bán hàng hoặc thương mại điện tử.
- **Thiếu ràng buộc**: Không yêu cầu giữ nguyên logic, không đổi tên hàm, không sửa thuật toán, không tự suy diễn nghiệp vụ ngoài dữ liệu đã cho.
- **Thiếu định dạng đầu ra**: Không yêu cầu Javadoc chuẩn, comment inline, bảng map biến hay code block Java.

Vì vậy, AI dễ trả lời chung chung, không khớp nghiệp vụ và có nguy cơ hiểu sai ý nghĩa của các biến `a`, `b`, `c`.

## 2. Prompt tối ưu mới

```text
Đóng vai Senior Java Developer chuyên bảo trì mã nguồn legacy.

Mục tiêu:
Phân tích hàm Java cũ dùng để tính chiết khấu đơn hàng, giải thích lại logic theo ngôn ngữ nghiệp vụ và bổ sung Javadoc/comment để đội bảo trì hiểu rõ mà không làm thay đổi hành vi chương trình.

Ngữ cảnh:
Đây là hệ thống E-commerce. Hàm cD được developer cũ viết để tính số tiền giảm giá cho đơn hàng. Code hiện tại dùng tên biến khó hiểu:
- a là tổng giá trị đơn hàng
- b là hạng thành viên:
  - b = 1 tương ứng Silver
  - b = 2 tương ứng Gold
  - giá trị khác tương ứng khách thường
- c là trạng thái có voucher freeship/ưu đãi bổ sung hay không

Ràng buộc:
- Không thay đổi logic xử lý ban đầu.
- Không đổi tên hàm nếu không cần thiết.
- Không tự thêm nghiệp vụ mới ngoài thông tin được cung cấp.
- Phải tạo bảng mapping biến kỹ thuật sang ý nghĩa nghiệp vụ.
- Phải sinh Javadoc chuẩn cho hàm.
- Phải thêm comment inline cho các dòng điều kiện hoặc phép tính phức tạp.
- Nếu có điểm chưa chắc chắn, phải ghi chú là giả định, không được khẳng định tuyệt đối.

Định dạng đầu ra:
1. Bảng mapping biến.
2. Giải thích ngắn gọn logic nghiệp vụ.
3. Mã nguồn Java đã có Javadoc và comment inline, đặt trong code block Java.

Đoạn code cần phân tích:

public class DiscountService {
    public double cD(double a, int b, boolean c) {
        double d = 0;

        if (a <= 0) {
            return 0;
        }

        if (b == 1) {
            d = a * 0.05;
        } else if (b == 2) {
            d = a * 0.10;
        } else {
            d = a * 0.02;
        }

        if (a > 1000000) {
            d += a * 0.03;
        }

        if (c) {
            d += 30000;
        }

        if (d > a * 0.2) {
            d = a * 0.2;
        }

        return d;
    }
}
```

## 3. Kết quả chạy thử Prompt với AI

### 3.1. Bảng mapping biến

| Biến    | Ý nghĩa kỹ thuật         | Ý nghĩa nghiệp vụ                                    |
| ------- | ------------------------ | ---------------------------------------------------- |
| `a`     | Giá trị kiểu `double`    | Tổng giá trị đơn hàng                                |
| `b`     | Giá trị kiểu `int`       | Hạng thành viên                                      |
| `b = 1` | Nhánh điều kiện thứ nhất | Thành viên Silver                                    |
| `b = 2` | Nhánh điều kiện thứ hai  | Thành viên Gold                                      |
| `c`     | Giá trị kiểu `boolean`   | Có sử dụng voucher freeship/ưu đãi bổ sung hay không |
| `d`     | Giá trị kiểu `double`    | Tổng số tiền được giảm                               |

---

### 3.2. Giải thích logic nghiệp vụ

Hàm `cD` dùng để tính tổng số tiền chiết khấu cho một đơn hàng trong hệ thống E-commerce.

Logic chính:

- Nếu tổng giá trị đơn hàng nhỏ hơn hoặc bằng 0, hệ thống không áp dụng chiết khấu.
- Nếu khách hàng là thành viên **Silver** (`b = 1`), giảm 5% giá trị đơn hàng.
- Nếu khách hàng là thành viên **Gold** (`b = 2`), giảm 10% giá trị đơn hàng.
- Nếu không thuộc Silver hoặc Gold, khách hàng thường được giảm 2%.
- Nếu đơn hàng lớn hơn 1.000.000, cộng thêm 3% ưu đãi.
- Nếu có voucher freeship hoặc ưu đãi bổ sung (`c = true`), cộng thêm 30.000 vào tổng số tiền giảm.
- Tổng chiết khấu tối đa không vượt quá 20% giá trị đơn hàng.

## 4. Mã nguồn Java đã có Javadoc và comment

```java
public class DiscountService {

    /**
     * Calculates the discount amount for an order based on order value,
     * membership level, and additional voucher status.
     *
     * Business rules:
     * - If the order amount is less than or equal to 0, no discount is applied.
     * - Silver members receive a 5% discount.
     * - Gold members receive a 10% discount.
     * - Regular customers receive a 2% discount.
     * - Orders greater than 1,000,000 receive an additional 3% discount.
     * - If a freeship or additional voucher is applied, 30,000 is added to the discount.
     * - The final discount must not exceed 20% of the order amount.
     *
     * Variable mapping:
     * - a: total order amount
     * - b: membership level, where 1 = Silver, 2 = Gold, other values = Regular customer
     * - c: whether the customer has a freeship/additional voucher
     *
     * @param a the total order amount
     * @param b the membership level; 1 represents Silver, 2 represents Gold,
     *          and other values represent regular customers
     * @param c true if a freeship or additional voucher is applied; false otherwise
     * @return the calculated discount amount
     */
    public double cD(double a, int b, boolean c) {
        double d = 0;

        // No discount is applied if the order amount is invalid.
        if (a <= 0) {
            return 0;
        }

        // Apply base discount according to customer membership level.
        if (b == 1) {
            // Silver member receives 5% discount.
            d = a * 0.05;
        } else if (b == 2) {
            // Gold member receives 10% discount.
            d = a * 0.10;
        } else {
            // Regular customer receives 2% discount.
            d = a * 0.02;
        }

        // Apply an additional 3% discount for high-value orders.
        if (a > 1000000) {
            d += a * 0.03;
        }

        // Add fixed discount amount when a freeship/additional voucher is applied.
        if (c) {
            d += 30000;
        }

        // Ensure the total discount does not exceed 20% of the order amount.
        if (d > a * 0.2) {
            d = a * 0.2;
        }

        return d;
    }
}
```

## 5. Kết luận

Prompt mới hiệu quả hơn prompt cũ vì đã áp dụng đủ 5 thành phần:

- **Vai trò**: Senior Java Developer chuyên bảo trì legacy code.
- **Mục tiêu**: Phân tích logic, giải thích nghiệp vụ, sinh Javadoc và comment.
- **Ngữ cảnh**: Hệ thống E-commerce, hàm tính chiết khấu đơn hàng.
- **Ràng buộc**: Không đổi logic, không tự thêm nghiệp vụ, phải map biến rõ ràng.
- **Định dạng**: Có bảng mapping, giải thích logic và code Java trong code block.

Nhờ đó, AI có thể tạo đầu ra rõ ràng, đúng nghiệp vụ, dễ bảo trì và giảm rủi ro gây lỗi khi chỉnh sửa mã nguồn cũ.
